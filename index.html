Regla general
Usa siempre transition: all var(--dur-fast) var(--ease-out) para hover/focus. Reserve --ease-spring para elementos que "aparecen" (modales, tooltips, chips que entran). Nunca animes sin duración explícita.

Acción principal del sistema. En NEO el botón primario es una superficie elevada con gradiente navy → indigo que se hunde al presionar.

Forms

Componentes de captura de datos del sistema NEO. Regla de relieve: los campos de entrada son pozos hundidos (--neu-in) porque reciben contenido; los controles accionables —botones, toggles, handles, steppers— son superficies elevadas (--neu-out). Anatomía base Figma: campo de 48px, radius 8px (NEO lo eleva a 12px), padding horizontal 16px, label 4px por encima, tipografía Nunito 16/24.

Glass Effect Recipe

$12,4M
RESERVAS · CAPITAL · PRIMAS
Superficie Glass
backdrop-filter:blur(12px) + borde de 1px blanco al 50%, sin sombra. El contorno define el panel sin elevarlo del fondo.
.glass-surface {
  background: rgba(232, 236, 244, 0.72);     /* --glass-bg      */
  backdrop-filter: blur(12px);
  -webkit-backdrop-filter: blur(12px);
  border: 1px solid rgba(255, 255, 255, 0.60); /* --glass-border  */
  box-shadow:
    inset 0 1px 0 rgba(255,255,255,0.95),       /* --glass-shine   */
    var(--neu-out-sm);                           /* elevación suave */
  border-radius: 16px;
}

/* Variante plana — sin borde, brillo ni sombra (superficie integrada) */
.glass-surface--flat {
  border: 1px solid rgba(255,255,255,0.5);
  box-shadow: none;
}

/* IMPORTANTE: requiere fondo con variación (gradiente o imagen)
   No visible sobre fondo plano --bg #e8ecf4 */
Cuándo usar Glass
Modales sobre contenido con gradiente · Overlays sobre dashboards con gráficas · Panels flotantes en pantallas con background imagen. No usar sobre fondo plano --bg — el blur no es visible.

Color Usage Guidelines

SÍ hacer
  • --navy para títulos principales y valores importantes
  • --indigo para elementos interactivos y brand
  • --text-sub para texto secundario y labels
  • --text-dim para metadatos y timestamps
  • Semánticos (green/yellow/red) solo para estados
NO hacer
  • Mezclar más de 2 colores brand en un componente
  • --red para estados no-error
  • --orange como color de acción (solo acento visual)
  • Texto --text-dim en tamaño menor a 11px
  • Usar --navy directo sobre --bg en texto pequeño (bajo contraste)
